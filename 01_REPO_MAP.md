## 5. Web Component (docs/)

### Overview
The repository includes a **static documentation website** hosted via GitHub Pages (Jekyll).  
Live URL: `https://codeshub.diwas.info.np/`

### Technology Stack
- **Theme**: Cayman (`pages-themes/cayman@v0.2.0`)
- **Plugins**: `jekyll-relative-links`
- **Site config**: `_config.yml` (defines title, plugins, supported languages)
- **Static site**: Plain HTML/CSS/JS in `docs/`
- **Dynamic data**: `docs/site_data.json` — generated by Python script, contains language files + commit history

### docs/ Contents
| File | Purpose |
|---|---|
| `index.html` | Main site page — displays language cards, recent commits |
| `css/style.css` | Site styling |
| `js/main.js` | Frontend interactivity (modal popups, dynamic content) |
| `site_data.json` | Auto-generated data: languages, files, sizes, commit history |
| `languages.md` | Jekyll-rendered language listing page |
| `CNAME` | Custom domain configuration |
| `README.md` | Documentation for the docs site itself |
| `scripts/generate_site_data.py` | Python script that scans repo and generates site_data.json |

### Update Mechanism
The `site_data.json` is **NOT manually maintained**. It is auto-generated:
1. **GitHub Actions workflow** (`.github/workflows/generate_docs.yml`) runs on:
   - Every push to `main` branch (when code files change)
   - Daily at midnight (cron schedule)
   - Manual trigger (`workflow_dispatch`)
2. **Python script** (`docs/scripts/generate_site_data.py`) scans the repo:
   - Identifies language directories (excludes `.git`, `docs`, `.github`)
   - Recursively finds code files by extension (`.py`, `.js`, `.java`, `.c`, `.cpp`, `.cs`, `.go`, `.rs`, `.php`, `.html`, `.ts`, `.sh`)
   - Gathers file metadata (size, line count, last modified)
   - Fetches last 5 git commits
3. Workflow commits updated `site_data.json` back to the repo

## 6. CI/CD

### GitHub Actions Workflow: `generate_docs.yml`

**Trigger**: Push to `main` when any language folder changes, plus daily cron and manual dispatch

**Skip conditions**: Commit messages containing `[skip ci]` or `[ci skip]`

**Steps**:
1. Checkout repo (full history for git commands)
2. Set up Python 3.10
3. Run `python docs/scripts/generate_site_data.py`
4. Commit and push `docs/site_data.json` if changed

**Permissions**: `contents: write` (for auto-committing)

**No other workflows**: No test, lint, build, or deployment workflows found.

## 9. Risk Areas

**NONE CRITICAL** — This is a pure code snippets repository with no:
- Executable builds or deployments
- Database connections
- External API dependencies
- Authentication/authorization systems
- User-generated content

**Minor considerations**:
- `Javascript/fastify/` sub-project has dependencies (`fastify`, `nanoid`, `tap`) — listed in `package.json` but not used by the main repo
- `guide/configure.txt` contains Postfix email server configuration — appears unrelated to the project (possible leftover/accidental commit)
- `docs/scripts/generate_site_data.py` uses `git` via subprocess — will fail gracefully if git is unavailable
