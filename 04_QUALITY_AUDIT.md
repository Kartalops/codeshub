# 04_QUALITY_AUDIT — codeshub

## Audit Overview
This is a code snippets repository — not a traditional web app. Quality focus is on:
- docs/ website correctness and usability
- Code snippet organization and naming
- CONTRIBUTING.md and README.md clarity
- Broken links or outdated references
- The Jekyll-generated static site

## Findings

### 1. `guide/configure.txt` — Unrelated File
**Risk:** LOW | **Area:** anomaly
**Finding:** `guide/configure.txt` contains Postfix email server configuration — completely unrelated to a code snippets platform. Likely an accidental commit.
**Action:** Consider removing or relocating to appropriate project.
**Note:** DO NOT touch without checking with maintainer — may have been intentional.

### 2. `typescript/` Directory Naming Inconsistency
**Risk:** LOW | **Area:** structure
**Finding:** All other language directories use capitalized names (`C/`, `Cpp/`, `Java/`, `Javascript/`, `Python/`) but `typescript/` uses lowercase.
**Impact:** Could cause inconsistency in the `generate_site_data.py` script or Jekyll configuration.
**Action:** Rename to `TypeScript/` to match convention. Low risk but touches many things.

### 3. `Javascript/fastify/` Sub-Project Has Unused Dependencies
**Risk:** LOW | **Area:** dependency
**Finding:** `Javascript/fastify/package.json` lists `fastify`, `nanoid`, and `tap` as dependencies. These are NOT used in the main CodesHub platform — they appear to be a sample sub-project someone contributed.
**Action:** No action needed for contribution purposes — this is a known oddity.

### 4. `docs/site_data.json` Auto-Generation Race Condition
**Risk:** LOW | **Area:** CI
**Finding:** The GitHub Actions workflow auto-generates `docs/site_data.json` on every push. If someone manually edits `site_data.json`, their changes will be overwritten.
**Action:** Add a comment at the top of the file warning: "DO NOT EDIT MANUALLY — Auto-generated by `.github/workflows/generate_docs.yml`"
**Note:** This is a docs-only fix, no code risk.

### 5. CONTRIBUTING.md — Vague Directory Structure
**Risk:** LOW | **Area:** docs
**Finding:** CONTRIBUTING.md lists example directory structure but doesn't include all current folders (e.g., `typescript/`, `Rust/`, `Bash/` mentioned but `Bash/` doesn't exist yet).
**Action:** Update CONTRIBUTING.md to reflect actual current structure and clarify naming conventions (capitalization).

### 6. `docs/` Website — Missing Language Card Detail
**Risk:** LOW | **Area:** UX
**Finding:** The docs website shows language cards but clicking a language card doesn't link to the GitHub source. Issue #352 tracks this.
**Action:** PR candidate — add source links to language cards.

### 7. `docs/README.md` — Outdated Reference
**Risk:** LOW | **Area:** docs
**Finding:** `docs/README.md` may reference the old site structure. Need to verify if current docs/README.md is accurate.
**Action:** Read `docs/README.md` — if outdated, update.

### 8. No Code Quality Standards
**Risk:** N/A | **Area:** process
**Finding:** No linter, no formatter, no test framework. Since this is a snippets repo, this is acceptable.
**Action:** None — not a PR candidate.

### 9. `docs/site_data.json` Format — Missing Field Documentation
**Risk:** LOW | **Area:** docs
**Finding:** The Python script `generate_site_data.py` generates `site_data.json` but the schema (what each field means) is not documented.
**Action:** Add a comment header to `docs/scripts/generate_site_data.py` explaining the JSON output schema.

### 10. README.md — Could Use Badges
**Risk:** LOW | **Area:** docs
**Finding:** README.md has no badges (CI status, license, GitHub stars). Other repos typically have these.
**Action:** Add standard badges: license, GitHub issues, Hacktoberfest.
